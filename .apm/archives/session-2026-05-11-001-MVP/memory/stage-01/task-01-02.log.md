# Task 1.2 - SQLite schema + db helpers

## Summary

`app/db.py` and `tests/test_db.py` were originally implemented against a schema doc that required `started_at` and `duration_minutes` to be NOT NULL, forcing provisional sentinel values on INSERT. This follow-up (superseding the prior log) replaces those workarounds with proper NULL semantics now that the schema doc is corrected.

## Details

### Original implementation (commit `764f5ff`)

`app/db.py` was written with `started_at TEXT NOT NULL` and `duration_minutes INTEGER NOT NULL` in the CREATE TABLE statement. To satisfy these constraints, `insert_pending_session` wrote `_now_utc()` for `started_at` and `0` for `duration_minutes` at INSERT time, with the intent that subsequent update helpers would overwrite them. This was lossy: sessions that never reached `active` or had a duration selected would carry sentinel values indistinguishable from real ones in stats queries.

The original implementation included:
- `init_db(path)` — opens/creates the DB, runs `CREATE TABLE IF NOT EXISTS` and both `CREATE INDEX IF NOT EXISTS`, commits, returns the connection.
- `DB_PATH` — module-level constant from `app.config.TEAMODE_DB_PATH`.
- `_now_utc()` — private helper returning an ISO-8601 UTC timestamp string.
- Ten write helpers with keyword-only arguments.
- All queries use `?` parameterised placeholders.
- `reconcile_crashed_sessions` builds the `IN (?,?,?,?)` clause dynamically from `_NON_TERMINAL_STATUSES`.

### Schema correction (commit `cfa7572`)

`docs/sqlite-schema.md` was updated to reflect that both columns are nullable, matching the stated semantics: `started_at` is NULL until the `active` transition; `duration_minutes` is NULL until the facilitator picks a length.

### Follow-up changes (commit `b96b0bd`)

**`app/db.py`:**
- Dropped `NOT NULL` from `started_at` and `duration_minutes` in `_CREATE_SESSIONS`, matching `docs/sqlite-schema.md` exactly.
- Removed `started_at` and `duration_minutes` from the `INSERT` in `insert_pending_session`; SQLite defaults both to NULL.
- Updated the docstring to reflect NULL semantics.

**`tests/test_db.py`:**
- `test_insert_pending_session_row_state`: assertions for `started_at` and `duration_minutes` now expect `None`; added explicit `duration_minutes` check.
- `test_update_duration`: added pre-update assertion confirming `duration_minutes` is `None` on a fresh insert; post-update asserts it equals the supplied value (25).
- `test_update_started_at_active` and `test_update_started_at_active_defaults_to_now`: added pre-update assertions confirming `started_at` is `None`; post-update assertions unchanged.
- `test_schema_init_idempotent_same_conn`: hardcoded schema SQL updated to match nullable columns.
- `test_schema_nullable_columns` (new): uses `PRAGMA table_info` to assert `notnull=0` for both `started_at` and `duration_minutes`.

Total tests: 29 (was 28; one added).
