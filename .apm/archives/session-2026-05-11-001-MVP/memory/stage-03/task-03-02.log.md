The previous worker introduced a call to `self.tree.process_application_commands(interaction)` which is not a valid `CommandTree` method (pyright error). Fixed by removing the block; application-command routing is handled by discord.py internals before `on_interaction` fires.

## Important Findings

`discord.ui.Label.component` is typed as `Item[Unknown]` (not as `TextInput`), which causes pyright to reject `.value` access. The workaround is `cast(discord.ui.TextInput[discord.ui.Modal], label.component)`. This pattern should be used in any future modal that reads TextInput values via a Label wrapper.

The `CommandTree` in discord.py 2.6 does not have a public `process_application_commands` method. Application-command interactions are routed internally in `ConnectionState.parse_interaction_create` before the `interaction` event is dispatched to user code. The `on_interaction` handler should not attempt to re-dispatch them.
