# Task 5.3 - Crash Reconciliation — Verify Wiring and Reorganise Tests

## Summary

Verified the startup ordering invariant in `teamode.py` and added the required inline comment. Reorganised the five reconciliation tests from `tests/test_db.py` into the new `tests/test_db_reconciliation.py` module, lifting the `conn` fixture to `tests/conftest.py`.

## Details

**Part A — Startup ordering verification:**

`teamode.py` was confirmed correct:
1. `db.init_db(conn)` called at line 22 — before reconciliation.
2. `db.reconcile_crashed_sessions(conn)` called at line 24 — before `bot.run(DISCORD_BOT_TOKEN)` at line 29.
3. Log message matches exactly: `"Reconciled %d crashed session(s) on startup"`.

Added the two-line ordering invariant comment immediately above the `reconcile_crashed_sessions` call as specified.

**Part B — Test reorganisation:**

- The `conn` fixture was in `test_db.py` only (not in `conftest.py`). Lifted it to `tests/conftest.py` alongside the `import sqlite3`, `import pytest`, and `from app.db import init_db` that it needs. Added a `# noqa: E402` comment since the import follows the `os.environ.setdefault` call.
- Created `tests/test_db_reconciliation.py` with the five reconciliation tests, the `_insert_with_status` helper, `_fetch` helper (duplicated locally — `_fetch` is also used by non-reconciliation tests in `test_db.py`, so it could not be moved without breaking those), `_NON_TERMINAL`/`_TERMINAL` constants, and a `from app.db import reconcile_crashed_sessions` import.
- Removed from `tests/test_db.py`: the `conn` fixture, the `reconcile_crashed_sessions` import, the `pytest` import (now unused), and the entire reconciliation section including the `_insert_with_status` helper, `_NON_TERMINAL`/`_TERMINAL` constants, and the five test functions, plus the `# reconcile_crashed_sessions` section header.
- `ruff format` reformatted `test_db.py` after the removals (trailing blank line cleanup).

**Fixture strategy decision:** `_fetch` was duplicated in the new module rather than moved to `conftest.py` because it is also used by ~20 non-reconciliation tests in `test_db.py`. Duplicating it produces a smaller diff than lifting it.
