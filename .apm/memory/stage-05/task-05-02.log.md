# Task 5.2 - Solo Facilitator 5-Minute Grace

## Summary

Implemented the 5-minute solo-facilitator grace watchdog on branch `feat/solo-grace`. When the facilitator leaves voice with no other humans remaining, a watchdog arms; if the facilitator rejoins within 5 minutes it cancels; on timeout the session is torn down cleanly (message rewritten, countdown cancelled, voice disconnected, SQLite row updated to `cancelled`).

## Details

All work performed on `feat/solo-grace` (created from `main`). No deviations from Task Prompt instructions were found — the codebase matched the described interfaces exactly.

**Part A — Plumbing in `IntentionModal.on_submit` and `_run_and_followup`:**
- After `voice.connect()` succeeds, stash `voice_client` in `self._bot._voice_clients[session_id]`.
- Capture the `asyncio.create_task(_run_and_followup())` return value and stash it in `self._bot._countdown_tasks[session_id]`.
- Inside `_run_and_followup`, on normal completion pop both `_voice_clients` and `_countdown_tasks` (alongside the existing `_edit_states.pop`).

**Part B — Module-level constant:**
- Added `_SOLO_GRACE_SECONDS = 300` near the other timing constants.

**Part C — Solo-leave branch in `on_voice_state_update`:**
- Replaced the `# T5.2 seam` no-op comment with logic that arms `_run_solo_grace` via `asyncio.create_task` when `remaining` is empty and no watchdog is already pending for the session.

**Part D — Rejoin detection:**
- Added a join-detection block at the top of `on_voice_state_update` (before the leave-detection block). If `after.channel` is set and differs from `before.channel`, look up the session in the joined channel; if it matches the facilitator and a solo-grace task is pending, cancel and pop it.

**Part E — `_run_solo_grace` method:**
- Added the full watchdog coroutine to `TeaModeBot`. Uses `sleep_seconds` injection parameter (default `_SOLO_GRACE_SECONDS`) for test-driven timeout. On `CancelledError` returns immediately. On timeout: pop `countdown_task`, `voice_client`, `edit_state`, and self; cancel+await countdown; rewrite message; disconnect voice; call `mark_cancelled`. All four cleanup steps are individually exception-guarded.

**Part F — `__init__` additions:**
- Added `_voice_clients`, `_countdown_tasks`, and `_solo_grace_tasks` dicts.

**Part G — Tests (`tests/test_session_solo_grace.py`):**
- 10 test cases covering both watchdog and listener-side behaviour (full list below).
- One correction needed after initial run: `task.cancelled()` returns `False` immediately after `task.cancel()` because the event loop hasn't had a chance to propagate the cancellation; added `await asyncio.sleep(0)` in `test_facilitator_rejoins_cancels_watchdog` to yield to the loop.
- One pyright fix: `edit_state.message.edit` is typed as `function` (from `AsyncMock(spec=discord.Message)`) so `assert_awaited_once_with` triggered `reportFunctionMemberAccess`; suppressed with `# type: ignore[union-attr]`, consistent with patterns in other test files.
